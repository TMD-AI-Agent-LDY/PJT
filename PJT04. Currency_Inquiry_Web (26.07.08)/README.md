# 환율 조회 웹앱 (Exchange Rate Web App)

Frankfurter Open API와 Streamlit을 연동하여 실시간 주요국 통화 환율을 계산하고 조회 이력을 관리할 수 있는 대시보드입니다.

### 주요 기능

- **실시간 환율 연동**: Open API 기반 실시간 환율 정보 조회 및 반영
- **직관적인 통화 선택**: 가독성을 높이기 위해 국가별 통화명 병기 (`format_func` 활용)
- **동일 통화 알림 기능**: 불필요한 API 요청을 차단하는 예외 처리 내장
- **시각화 Metric 제공**: 변환 총액 및 단일 환율 수치를 직관적인 카드 레이아웃으로 표시
- **누적 히스토리 트래킹**: `st.session_state`를 통해 대시보드 하단에 이전 조회 이력을 실시간 누적 관리

---

### 프로그램 코드 동작 설명 (Code Flow)

본 프로그램은 Streamlit의 실행 매커니즘과 Python의 네트워크 통신 라이브러리를 흐름에 따라 아래와 같이 처리합니다.

1. **세션 상태(Session State) 관리**
   - Streamlit은 웹페이지 내 위젯의 상태가 변경될 때마다 전체 코드가 처음부터 다시 실행(Re-run)되는 특징이 있습니다.
   - 조회 이력이 초기화되는 것을 막기 위해 `st.session_state`에 `history` 리스트를 등록하여 웹 브라우저가 유지되는 동안 데이터를 영구적으로 보존합니다.

2. **통화 매핑 및 UI 렌더링**
   - 단순한 통화 코드(`USD`, `KRW`) 대신 가독성을 높이기 위해 명칭 딕셔너리를 선언했습니다.
   - `st.selectbox`의 `format_func` 내부에 람다 함수(`lambda x: currency_names[x]`)를 적용함으로써 내부 연산은 표준 통화 코드로 처리하되, 화면에는 친숙한 국가별 명칭이 노출되도록 동적 매핑했습니다.

3. **조건별 분기 및 예외 처리**
   - 사용자가 조회 버튼을 누르면 무조건 API를 호출하지 않고, 기준 통화와 변환 통화가 일치하는지 먼저 검사합니다.
   - 두 통화가 같다면 외부 네트워크 요청을 차단하고 `st.warning` 안내 문구를 출력하여 서버 자원과 네트워크 트래픽을 아낍니다.

4. **API 통신 및 JSON 데이터 파싱**
   - 통화가 다를 경우 `requests.get()` 함수를 통해 Frankfurter API 서버로 실시간 환율 조회를 요청합니다.
   - 외부 인프라 장애나 네트워크 단절에 대응할 수 있도록 `try-except` 블록으로 통신 과정을 캡슐화하였으며, HTTP 응답 상태 코드가 `200(OK)`인 정상 상황에서만 JSON 바디를 디코딩하여 필요한 환율 정보(`rates`)를 추출합니다.

5. **컴포넌트 렌더링 및 이력 누적**
   - 연산이 완료되면 사용자가 직관적으로 수치를 파악할 수 있도록 대시보드 전용 시각화 컴포넌트인 `st.metric`을 사용하여 환율 카드를 렌더링합니다.
   - 조회가 성공할 때마다 해당 내역을 세션 히스토리에 담고, 화면 하단에 출력할 때는 `reversed()` 처리를 통해 가장 최근에 조회한 내역이 최상단에 배치되도록 타임라인 역순 구조를 구현했습니다.

### 설치 및 실행 방법

1. 필요한 라이브러리를 설치합니다.

```bash
pip install streamlit requests
```

2. 소스코드가 저장된 디렉토리로 터미널을 통해 이동합니다.
3. 아래 명령어를 입력하여 앱을 실행합니다.

```bash
streamlit run app.py
```
